# 한계 & 향후 과제

> 미해결 사항을 명확히 하고, 프로젝트가 실제 한계를 인식하고 있음을 보여줍니다.

## 법률 검토 필요

이 가이드는 법률 자문이 아닙니다. 다음 사항은 전문가 법률 검토가 필요합니다.

- 감사 로그의 규정 최소 보존 기간
- 동의 요건 및 정보 주체의 권리 (한국 개인정보보호법)
- 국경 간 데이터 전송 허용 조건
- 보안 침해 발생 시 통보 의무와 기한
- 의료 기록 원본 보존 의무 기간 (한국 의료법)
- 연구 목적 데이터 활용 시 환자 동의 요건

## 운영 런북 미비

보안 운영, 사고 대응, 키 교체, 로그 검토, 비상 접근 절차는 별도의 운영 런북으로 문서화되어야 합니다.

현재 이 가이드는 아키텍처 수준의 설계를 다루며, 실제 운영 절차는 향후 과제입니다.

## 침투 테스트 미실시

현재 가이드는 아키텍처 수준입니다. 침투 테스트 및 애플리케이션 수준 보안 테스트는 향후 과제입니다.

## 비용 및 확장성 평가 미비

다음 항목은 비용 및 성능 관점에서 평가가 필요합니다.

- KMS API 호출 비용 (대규모 서명 운영)
- VPC Interface Endpoint 비용 (소규모 환경)
- Schema-per-Tenant 스케일링 (테넌트 수 증가 시)
- S3 Object Lock Compliance 모드 운영 비용

## 커넥션 풀링과 테넌트 컨텍스트 검증

Lambda, 데이터베이스 커넥션 풀링, Schema-per-Tenant 설계, 테넌트 컨텍스트 간의 상호작용은 신중하게 테스트되어야 합니다. 특히 다음 사항이 미해결 상태입니다.

- RDS Proxy와 Schema-per-Tenant의 호환성
- 커넥션 풀 내 `search_path` 및 세션 변수 격리
- 커넥션 재사용 시 테넌트 컨텍스트 오염 가능성

## 제3자 보안 도구 비교 미포함

일부 컨트롤은 AWS 네이티브 서비스 또는 제3자 도구로 구현할 수 있습니다. 이 가이드는 현재 AWS 네이티브 설계를 우선합니다. 제3자 솔루션과의 비교는 향후 과제입니다.

## 재해 복구 및 백업 전략 미비

백업, 복원, 재해 복구, 멀티 리전 전략은 별도로 개발되어야 합니다.

## RLS 성능 영향 미평가

Row-Level Security 정책이 복잡한 쿼리 또는 대규모 데이터에서 성능에 미치는 영향은 실제 부하 테스트를 통해 검증되어야 합니다.

## 향후 개발 우선순위 (초안)

| 우선순위 | 항목 |
|---------|------|
| 1 | 법률 검토 항목 명확화 및 전문가 자문 |
| 2 | 커넥션 풀링 + Schema-per-Tenant 호환성 테스트 |
| 3 | 비용 및 확장성 평가 |
| 4 | 운영 런북 초안 작성 |
| 5 | 침투 테스트 계획 수립 |
| 6 | 재해 복구 전략 개발 |
