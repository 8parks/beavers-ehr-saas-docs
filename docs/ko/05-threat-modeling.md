# 위협 모델링

어떤 공격자가, 어떤 경로로, 어떤 자산을 노릴 수 있는지를 먼저 파악해야 제대로 된 방어를 설계할 수 있습니다. 여기서의 보안 요구사항은 4장(컴플라이언스)에서 도출된 요구사항을 기준으로 합니다.

## 보안 목표 {#보안-목표}

이 플랫폼이 달성해야 하는 보안 목표입니다.

| 목표 | 설명 |
|------|------|
| 기밀성 (Confidentiality) | PHI가 인가된 사용자 외에는 접근되지 않아야 합니다. |
| 무결성 (Integrity) | 의료 기록이 무단으로 수정되지 않아야 합니다. |
| 가용성 (Availability) | 병원 운영에 필요한 서비스가 안정적으로 제공되어야 합니다. |
| 프라이버시 (Privacy) | 환자 개인정보가 목적 범위 내에서만 처리되어야 합니다. |
| 책임추적성 (Accountability) | 누가 무엇을 했는지 나중에 확인할 수 있어야 합니다. |

## 위협 모델링 {#위협-모델링}

### 위협 모델링 방법론

두 가지 방법론을 참고하여 위협을 식별합니다.

**STRIDE** — 기술적 위협 분류

| 위협 유형 | 설명 |
|---------|------|
| Spoofing | 다른 사용자 또는 서비스로 위장 |
| Tampering | 데이터 또는 시스템 무단 변조 |
| Repudiation | 행위를 부인할 수 있는 상황 |
| Information Disclosure | 민감 정보의 무단 노출 |
| Denial of Service | 서비스 가용성 침해 |
| Elevation of Privilege | 권한 상승 |

**LINDDUN** — 프라이버시 위협 분류

| 위협 유형 | 설명 |
|---------|------|
| Linkability | 서로 다른 데이터 간 연결을 통한 식별 |
| Identifiability | 개인 식별 가능성 |
| Non-repudiation | 개인이 부인할 수 없는 흔적 |
| Detectability | 특정 데이터 존재 여부 탐지 |
| Disclosure | 정보 노출 |
| Unawareness | 정보 주체가 인지하지 못한 처리 |
| Non-compliance | 규제 위반 |
