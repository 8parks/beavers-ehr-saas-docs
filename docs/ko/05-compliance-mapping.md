# 컴플라이언스 범위 & 규제 매핑

> 고려된 규정과 표준, 우선순위 기준, 아키텍처 설계에 미치는 영향을 설명합니다.

::: warning 면책 조항
이 가이드는 법률 자문을 제공하지 않으며 완전한 규제 컴플라이언스를 주장하지 않습니다. 레퍼런스 아키텍처와 관련된 선택된 규제 및 보안 요구사항을 요약하고 기술적 설계 고려사항으로 번역하는 것이 목적입니다.
:::

## 규제 분석의 목적

이 섹션은 다음을 목적으로 합니다.

- 관련 규정 및 표준 식별
- 아키텍처 결정에 영향을 미치는 핵심 요구사항 파악
- 규제 요구사항을 기술적 설계 고려사항으로 번역
- 미해결 법적/운영적 질문 식별

## 적용 가능한 규정 및 표준

| 규정/표준 | 주요 관심사 | 우선순위 |
|-----------|-----------|---------|
| HIPAA | PHI 보호, 접근 제어, 감사 로그, 보안 사고 대응 | 높음 |
| HITECH | HIPAA 강화, 침해 통보 의무 | 높음 |
| 개인정보보호법 (한국) | 개인정보 처리, 가명처리, 동의 | 높음 |
| 의료법 (한국) | 의료 기록 보존, 접근 권한 | 높음 |
| ISO/IEC 27001 | 정보보안 관리 체계 | 중간 |
| AWS Well-Architected 보안 기반 | AWS 네이티브 보안 모범 사례 | 높음 |
| OWASP ASVS | 보안 검증 참고 기준 | 중간 |

## 규제-아키텍처 매핑

| 요구사항 범주 | 설계 고려사항 | AWS 서비스 | 관련 시나리오 | 비고 |
|--------------|-------------|-----------|--------------|------|
| 접근 제어 | 역할 기반, 테넌트 인식 접근 제어 | Cognito, IAM, Lambda, RDS/Aurora | 환자 기록 조회 | 애플리케이션 레이어 검증 필수 |
| 감사 추적성 | 관리자 및 PHI 접근 이벤트 기록 | CloudTrail, CloudWatch, S3, Athena | 감사 로그 무결성 | 보존 기간 법률 검토 필요 |
| 암호화 | PHI 및 비밀 정보 암호화 | KMS, Secrets Manager, RDS/Aurora, S3 | 기록 생성 및 저장 | 키 소유권 모델 명확화 필요 |
| 가명처리 | 연구 데이터 가명처리 | Lambda, KMS, DynamoDB | 연구 데이터셋 생성 | 법률 기준 충족 여부 검토 필요 |

## 미해결 법적/운영적 질문

- 각 규제에서 요구하는 최소 감사 로그 보존 기간
- 연구 목적 데이터 활용 시 환자 동의 요건 (한국 개인정보보호법 기준)
- 국경 간 데이터 전송 허용 조건
- 보안 침해 발생 시 통보 의무와 기한
- 의료 기록 원본 보존 의무 기간 (한국 의료법 기준)
