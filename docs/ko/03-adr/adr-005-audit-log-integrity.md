# ADR-005. 감사 로그 보존 및 무결성

## 상태

제안됨

## 컨텍스트

EHR 시스템은 강력한 감사 추적성을 요구합니다. PHI 접근, 관리자 활동, 보안 이벤트에 대한 로그를 수집하고, 장기 보존하며, 변조를 방지해야 합니다. 어떤 AWS 서비스 조합으로 이를 달성할지 결정해야 합니다.

## 결정

<!-- 작성 예정: 선택된 감사 로그 보존 및 무결성 전략 -->

**초안:** CloudTrail + CloudWatch Logs로 이벤트를 수집하고, S3 Object Lock(WORM)으로 변조를 방지하며, KMS로 로그를 암호화합니다.

## 고려된 대안

| 대안 | 설명 |
|------|------|
| CloudWatch Logs만 사용 | 단순하지만 변조 방지 기능 없음 |
| CloudTrail + S3 | API 레벨 감사 가능, Object Lock으로 불변성 확보 |
| CloudTrail + S3 + Athena | 쿼리 가능한 감사 로그 분석 추가 |
| 외부 SIEM 연동 | 외부 시스템 의존성 추가 |

## 보안 함의

- S3 Object Lock Compliance 모드 사용 시 보존 기간 전 삭제 불가
- KMS 암호화로 미사용 로그 보호
- CloudTrail CloudWatch Logs Integration으로 실시간 알람 가능
- 로그에 불필요한 PHI가 포함되지 않도록 주의

## 컴플라이언스 매핑

| 요구사항 범주 | 관련 설계 결정 |
|--------------|--------------|
| 감사 추적성 | CloudTrail, CloudWatch, S3 Object Lock |
| 데이터 무결성 | S3 Object Lock, KMS 서명 |
| 데이터 보존 | S3 보존 정책, Object Lock |

## 미해결 질문

- 규제상 요구되는 최소 로그 보존 기간 (법률 검토 필요)
- 감사 로그 접근 권한 분리 방법 (운영자 vs 감사자)
- Amazon Athena를 통한 로그 쿼리 접근 제어 설계
- 로그에 포함 가능한 식별 정보의 범위 (PHI 여부 판단)
