# ADR-009. 연구 데이터셋 생성 및 가명처리

## 상태

제안됨

## 컨텍스트

EHR 시스템에서 수집된 PHI는 연구 목적으로 활용될 수 있습니다. 그러나 연구 목적으로 데이터를 공유할 때는 개인 식별 가능 정보가 적절히 처리되어야 합니다. 가명처리(pseudonymization) 또는 비식별화(de-identification) 방법을 어떻게 설계할지 결정해야 합니다.

## 결정

<!-- 작성 예정: 선택된 가명처리 전략 -->

**초안:** 연구 데이터셋 생성 시 환자 식별자를 KMS 기반 해시 또는 별도 가명 매핑 테이블로 처리하고, 생성 프로세스를 감사 가능한 Lambda 파이프라인으로 구현합니다.

## 고려된 대안

| 대안 | 설명 |
|------|------|
| 단순 필드 삭제 (De-identification) | 식별자 필드 제거, 재식별 위험 존재 |
| 가명처리 (Pseudonymization) | 식별자를 가명으로 대체, 역매핑 테이블 별도 보관 |
| AWS 완전 관리형 데이터 처리 | AWS Glue, Amazon Comprehend Medical 활용 |

## 보안 함의

- 가명 매핑 테이블은 PHI와 동등한 수준으로 보호 필요
- 연구 데이터셋 생성 프로세스 전체를 감사 로그로 기록
- 연구자 접근 권한은 최소 필요 원칙 적용
- 생성된 데이터셋의 재식별 위험 평가 절차 필요

## 컴플라이언스 매핑

| 요구사항 범주 | 관련 설계 결정 |
|--------------|--------------|
| 데이터 보호 | 연구 데이터의 가명처리, 재식별 방지 |
| 감사 추적성 | 데이터셋 생성 이력 기록 |
| 제3자 공유 | 가명처리 후 공유 기준 설계 |

## 미해결 질문

- 한국 개인정보보호법 가명처리 기준을 충족하는 구체적 방법론 선택 (법률 검토 필요)
- 가명 매핑 테이블의 보존 기간 및 삭제 정책
- 연구자가 가명처리된 데이터에서 역으로 개인을 식별하려 시도하는 경우의 탐지/방지
