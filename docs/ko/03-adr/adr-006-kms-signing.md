# ADR-006. KMS 기반 서명 및 암호화 전략

## 상태

제안됨

## 컨텍스트

EHR 시스템에서 의료 기록의 무결성 증명과 PHI 암호화는 핵심 보안 요구사항입니다. AWS KMS의 Sign/Verify API를 활용하여 의료 기록에 디지털 서명을 적용하고, KMS를 통한 암호화 전략을 어떻게 설계할지 결정해야 합니다.

<!-- 작성 예정: KMS 비대칭 키, CMK, 키 정책 관련 내용 -->

## 결정

<!-- 작성 예정: 선택된 KMS 서명 및 암호화 전략 -->

**초안:** KMS 비대칭 키(RSA 또는 ECDSA)를 사용하여 의료 기록 해시에 디지털 서명을 적용하고, KMS CMK로 PHI 저장 데이터를 암호화합니다.

## 고려된 대안

| 대안 | 설명 |
|------|------|
| 애플리케이션 레이어 서명 | 자체 PKI 또는 애플리케이션에서 키 관리 |
| KMS Sign/Verify API | AWS 관리 키 인프라 활용 (권장) |
| CloudHSM | 전용 HSM, 높은 비용 |

## 보안 함의

- KMS 키 정책으로 서명 키 접근을 특정 Lambda 역할로 제한
- 서명 검증 시 공개키 또는 KMS Verify API 활용
- 테넌트별 독립 KMS 키 사용 여부 결정 필요
- 키 교체(rotation) 정책 및 이전 서명 검증 방법 설계 필요

## 컴플라이언스 매핑

| 요구사항 범주 | 관련 설계 결정 |
|--------------|--------------|
| 암호화 | KMS 기반 저장 데이터 암호화 |
| 데이터 무결성 | KMS Sign/Verify로 의료 기록 서명 |
| 키 관리 | KMS CMK, 키 정책, 키 교체 |

## 미해결 질문

- 테넌트별 CMK 분리 여부 (비용 vs 격리 수준 트레이드오프)
- 키 교체 후 이전 키로 서명된 기록의 검증 방법
- 서명 키에 대한 비상 접근(break-glass) 절차
- KMS API 호출 비용이 대규모 서명 운영에서 적절한지 평가
