# Multi-tenant EHR SaaS 보안 아키텍처 가이드

> AWS 네이티브 멀티 테넌트 EHR SaaS를 위한 아키텍처 결정 기록 및 보안 설계 문서입니다.

## 목적

이 가이드는 가상의 AWS 네이티브 EHR(전자건강기록) SaaS 플랫폼을 위한 아키텍처 결정, 보안 설계 지침, 위협 모델링, 컴플라이언스 매핑을 체계적으로 정리한 문서 프로젝트입니다.

여러 병원 또는 의료 기관이 동일한 SaaS 플랫폼을 공유하는 멀티 테넌트 환경에서, 강력한 테넌트 격리, 감사 추적성, 컴플라이언스 인식 보안 컨트롤을 어떻게 설계할 수 있는지를 탐구합니다.

> 이 가이드는 선택된 규제 및 보안 요구사항을 AWS 네이티브 EHR SaaS 레퍼런스 모델의 아키텍처 수준 설계 고려사항으로 번역하는 것이 목표입니다.

## 범위

### 포함 범위

- 멀티 테넌트 EHR SaaS 레퍼런스 아키텍처
- 테넌트 격리 전략 및 계정 분리 전략
- ID 및 접근 관리
- PHI 데이터 보호
- 감사 로그 및 로그 무결성
- KMS 기반 암호화 및 서명
- 보안 이상 탐지 및 자동 대응 흐름
- 연구 데이터셋 생성 및 가명처리 흐름
- 위협 모델링 및 위협-컨트롤 매핑
- 컴플라이언스 인식 설계 매핑
- AWS 네이티브 보안 컨트롤 카탈로그
- OWASP ASVS 기반 검증 체크리스트
- 한국어 우선, 영어 지원 이중 언어 문서 구조

### 제외 범위

- 완전한 프로덕션 애플리케이션 구현
- 법적 컴플라이언스 보장
- 규정의 공식 법률 해석
- 완전한 침투 테스트
- 완전한 비용 최적화 분석

## 대상 독자

- AWS 기반 보안 아키텍처에 관심 있는 학생 및 개발자
- 헬스케어 SaaS 플랫폼의 멀티 테넌트 보안 설계를 탐구하는 엔지니어
- 컴플라이언스 인식 클라우드 아키텍처를 공부하는 분

## 핵심 설계 목표

| 목표 | 설명 |
|------|------|
| 멀티 테넌트 격리 | 테넌트 간 PHI 접근 방지, 스키마/서비스 레이어 격리 |
| PHI 보호 | 저장 및 전송 시 암호화, 키 관리, 안전한 인증/인가 |
| 감사 추적성 및 로그 무결성 | CloudTrail, S3 Object Lock, KMS 기반 로그 보존 |
| 컴플라이언스 인식 아키텍처 | HIPAA, 개인정보보호법, ISO 27001 등 연계 |
| 보안 자동화 및 대응 | AWS Config, Security Hub, EventBridge, SSM 자동화 |

## 아키텍처 한 눈에 보기

```
아키텍처 중심 → 위협 인식 → 컴플라이언스 연계 → 검증 준비
```

1. EHR SaaS 시스템 및 가정 정의
2. AWS 네이티브 레퍼런스 아키텍처 제시
3. 자산, 행위자, 신뢰 경계, 위협 식별
4. AWS 네이티브 컨트롤이 특정 위협을 완화하는 방법 설명
5. 보안 컨트롤을 컴플라이언스 요구사항에 매핑
6. 시나리오 기반 설계 지침 제공
7. 설계를 검증 체크리스트로 변환
