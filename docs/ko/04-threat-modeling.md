# 보안 요구사항 & 위협 모델링

> 자산, 위협 행위자, 위협 시나리오를 식별하고 AWS 보안 컨트롤에 매핑합니다.

## 위협 모델링 방법론

이 가이드에서는 다음 구성 요소를 기반으로 위협 모델링을 수행합니다.

- **자산 (Assets):** 보호해야 할 데이터 및 시스템 구성 요소
- **행위자 (Actors):** 위협을 가할 수 있는 내부 및 외부 행위자
- **신뢰 경계 (Trust Boundaries):** 신뢰 수준이 변경되는 경계
- **위협 시나리오 (Threat Scenarios):** 식별된 공격 벡터와 악용 가능한 취약점
- **보안 컨트롤 (Security Controls):** 위협을 완화하는 AWS 네이티브 컨트롤

## 위협 행위자

| 행위자 | 설명 | 동기 |
|--------|------|------|
| 외부 공격자 | 인터넷을 통한 무단 접근 시도 | 데이터 탈취, 랜섬웨어 |
| 악의적 테넌트 사용자 | 자신의 테넌트 권한을 넘어 타 테넌트 접근 시도 | 경쟁 정보 탈취 |
| 손상된 병원 관리자 | 관리자 계정이 침해된 경우 | 과도한 PHI 접근 |
| 플랫폼 운영자 실수 | 의도치 않은 잘못된 구성 | 사고, 서비스 장애 |
| 과도한 권한의 내부 관리자 | 최소 권한 원칙 위반 | 내부자 위협 |
| 잘못 구성된 클라우드 리소스 | 공개 S3 버킷 등 구성 오류 | 의도치 않은 데이터 노출 |

## OWASP ASVS 기반 보안 요구사항 매핑

OWASP ASVS를 구현 및 검증 참고 기준으로 활용합니다. 단, ASVS만으로 법적 컴플라이언스를 증명하지는 않습니다.

| ASVS 영역 | 설명 | 관련 설계 결정 |
|-----------|------|--------------|
| 아키텍처 및 위협 모델링 | 보안 아키텍처 설계 | 이 문서 전체 |
| 인증 | 안전한 인증 메커니즘 | Cognito, ADR-003 |
| 세션 관리 | 세션 토큰 보호 | JWT 토큰 설계 |
| 접근 제어 | 테넌트/역할 기반 인가 | ADR-001, ADR-003 |
| 암호학 | 암호화 및 서명 | ADR-006, KMS |
| 오류 처리 및 로깅 | 감사 이벤트 기록 | ADR-005 |
| 데이터 보호 | PHI 보호 | 여러 ADR |
| 통신 보안 | TLS, 프라이빗 연결 | ADR-008 |
| API 보안 | API Gateway 보안 | 시나리오 설계 |
| 안전한 구성 | 리소스 구성 표준 | ADR-007, AWS Config |
