# 검증 체크리스트

> 설계를 검증 가능한 체크리스트로 변환합니다.

## 상태 값 정의

| 상태 | 설명 |
|------|------|
| 제안됨 | 설계 결정으로 제안됨 |
| 설계됨 | 아키텍처 수준에서 설계 완료 |
| 구현됨 | 실제 구현 완료 |
| 테스트됨 | 검증 테스트 완료 |
| 검토 필요 | 추가 검토 또는 법률 자문 필요 |
| 범위 외 | 이 가이드의 범위를 벗어남 |

## 신원 및 인증

| ID | 요구사항 | 검증 방법 | AWS 증거 | 상태 |
|----|---------|---------|---------|------|
| AUTH-001 | 사용자는 PHI 접근 전 Cognito를 통해 인증해야 합니다. | Cognito Authorizer 구성 검토 | API Gateway 인가자 구성 | 제안됨 |
| AUTH-002 | MFA가 관리자 역할에 강제 적용되어야 합니다. | Cognito 정책 검토 | Cognito User Pool 설정 | 제안됨 |
| AUTH-003 | JWT 토큰 만료 정책이 적용되어야 합니다. | 토큰 만료 시간 검토 | Cognito App Client 설정 | 제안됨 |
