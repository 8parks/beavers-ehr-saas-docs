# S6. 운영자 비상 접근 (Break-glass)

일반적으로 플랫폼 운영자는 테넌트 PHI에 접근하지 않습니다. 하지만 심각한 장애나 보안 사고 상황에서 부득이하게 접근이 필요할 수 있습니다. 이 시나리오는 그 예외 상황을 어떻게 통제할 것인지 다룹니다.

## 비즈니스 흐름

비상 접근은 최후의 수단입니다. 일반적인 운영 경로가 모두 막혔을 때만 사용하며, 접근 시작부터 종료까지 모든 활동이 즉시 기록되고 검토되어야 합니다.

## 보안 목표

- 비상 접근은 승인된 인원만 사용할 수 있어야 합니다.
- 접근 시도, 승인, 실제 활동이 모두 실시간으로 기록되어야 합니다.
- 접근 후 즉각적인 검토 및 사후 보고가 이루어져야 합니다.
- 비상 접근 권한은 사용 후 즉시 회수됩니다.
