# S4. Tenant 온보딩

새 병원이 플랫폼에 합류할 때 어떤 과정을 거치는지, 그리고 그 과정에서 보안을 어떻게 유지하는지 다룹니다.

## 비즈니스 흐름

새 병원이 계약을 체결하면, 해당 병원을 위한 독립된 테넌트 환경이 만들어져야 합니다. 이때 데이터 격리, 초기 관리자 계정 생성, 암호화 키 설정 등이 함께 이루어집니다.

## 보안 목표

- 새 테넌트의 데이터가 기존 테넌트와 완전히 격리되어야 합니다.
- 초기 관리자 자격증명이 안전하게 전달되어야 합니다.
- 온보딩 과정 전체가 감사 로그로 기록되어야 합니다.
