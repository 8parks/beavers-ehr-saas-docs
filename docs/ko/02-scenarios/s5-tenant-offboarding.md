# S5. Tenant 오프보딩

병원이 플랫폼을 떠날 때 데이터를 어떻게 처리하고, 접근 권한을 어떻게 회수하며, 잔여 데이터를 어떻게 파기할 것인지 다룹니다.

## 비즈니스 흐름

계약 종료 시 해당 병원의 데이터를 반환하거나 파기하고, 모든 접근 경로를 차단해야 합니다. 규제에 따라 일정 기간 보존이 필요한 데이터는 별도로 처리합니다.

## 보안 목표

- 오프보딩 후 해당 테넌트 데이터에 접근할 수 없어야 합니다.
- 데이터 반환 또는 파기가 검증 가능한 방식으로 수행되어야 합니다.
- 접근 키, 자격증명, 암호화 키가 안전하게 폐기되어야 합니다.
