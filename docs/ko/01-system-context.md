# 시스템 컨텍스트 & 가정

> 보안, 컴플라이언스, 아키텍처를 논하기 전에 설계 대상 시스템을 정의합니다.

## 서비스 개요

Baby Beavers EHR SaaS는 여러 병원 또는 의료 기관이 공통 플랫폼 인프라를 공유하는 멀티 테넌트 SaaS 모델의 가상 전자건강기록 플랫폼입니다.

## 사용자 및 역할

| 역할 | 설명 | 테넌트 소속 |
|------|------|------------|
| 환자 (Patient) | 자신의 진료 기록 조회 가능 | 테넌트 (병원) |
| 의사 (Doctor) | 환자 기록 작성, 조회, 서명 | 테넌트 (병원) |
| 병원 관리자 (Hospital Admin) | 병원 내 사용자 관리, 설정 | 테넌트 (병원) |
| 연구자 (Researcher) | 가명처리된 연구 데이터셋 접근 | 테넌트 (병원) 또는 별도 |
| 시스템 관리자 (System Admin) | SaaS 플랫폼 수준 운영 | SaaS 제공자 |
| 보안 운영자 (Security Operator) | 보안 이상 탐지 및 대응 | SaaS 제공자 |

## 테넌트 모델

### 테넌트란 무엇인가?

이 설계에서 **테넌트는 병원 또는 의료 기관 단위**로 정의됩니다.

- 각 테넌트는 독립된 데이터 격리 영역을 가집니다.
- 테넌트 내 사용자는 동일 테넌트의 PHI에만 접근 가능합니다.

### 테넌트 계층

| 역할 유형 | 설명 |
|-----------|------|
| 테넌트 관리자 (Tenant Admin) | 병원 내 사용자 및 설정 관리 |
| 애플리케이션 사용자 (App User) | 의사, 간호사, 환자 등 실사용자 |
| 플랫폼 관리자 (Platform Admin) | SaaS 제공자 측 운영 관리자 |

### 크로스 테넌트 접근 가정

- 기본적으로 크로스 테넌트 PHI 접근은 **허용되지 않습니다**.
- 플랫폼 관리자도 테넌트 PHI에 직접 접근하지 않는 것을 원칙으로 합니다.

## 데이터 분류

| 데이터 유형 | 설명 | 보호 수준 |
|-------------|------|-----------|
| PHI (Protected Health Information) | 환자 기록, 의료 노트, 진단 정보 | 최고 |
| 가명처리 데이터 | 연구 목적 비식별화/가명처리 데이터 | 높음 |
| 감사 로그 | API 접근, 관리자 활동, 보안 이벤트 | 높음 |
| 구성 데이터 | 테넌트 설정, IAM 정책, 키 메타데이터 | 중간 |
| 공개/정적 데이터 | 공개 UI 자산, 공개 API 문서 | 낮음 |

## 신뢰 경계

<!-- 작성 예정: 신뢰 경계 다이어그램 및 설명 -->

주요 신뢰 경계:

- 인터넷 ↔ WAF/API Gateway (외부 경계)
- API Gateway/ALB ↔ Lambda (서비스 레이어 경계)
- Lambda ↔ RDS/Aurora (데이터 레이어 경계)
- 관리 계정 ↔ 멤버 계정 (계정 경계)
- 테넌트 A ↔ 테넌트 B (테넌트 격리 경계)

## 제외 범위

- 완전한 프로덕션 애플리케이션 구현
- 규제에 대한 법적 해석
- 침투 테스트 결과
- 완전한 DR(재해 복구) 계획
- 제3자 보안 솔루션 비교
