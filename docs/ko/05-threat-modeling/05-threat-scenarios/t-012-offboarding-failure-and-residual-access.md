# T-12 오프보딩 실패 및 잔존 접근

## 위협 개요

| 항목 | 내용 |
|------|------|
| 위험도 | 중간 |
| 관련 워크로드 | `S5`, `S3`, `S6` |
| 관련 자산 | `A-01`, `A-02`, `A-03`, `A-04`, `A-06`, `A-09` |
| 관련 경계 | `TB-02`, `TB-05`, `TB-06`, `TB-07`, `TB-08` |
| 대표 행위자 | 오프보딩 대상 사용자, 과거 연구 수신자, 운영자 |

오프보딩은 계정 삭제만으로 끝나지 않습니다. 세션, 데이터셋 전달 권한, dataset artifact, export, backup, cache가 남아 있으면 계약 종료 후에도 접근이 지속될 수 있습니다.

## 검토 대상

- Cognito 사용자 비활성화와 세션 회수 절차
- `tenant_registry.status` 전이 절차와 오프보딩 승인 기록
- 임상 파일 버킷과 연구 데이터셋 버킷의 tenant 관련 객체와 export 파일
- 데이터셋 요청·승인 상태 저장소의 잔존 request와 artifact 참조
- 활성화된 데이터셋 전달 권한, cache, backup 식별 정보, break-glass 예외 권한 회수 절차

## 공격 성립 조건

- 계정 비활성화와 세션 무효화가 분리되어 있지 않음.
- 데이터 파기 범위에 생성된 dataset, export, cache, 백업이 포함되지 않음.
- 오프보딩 완료 판단 기준과 기록이 없음.
- break-glass 또는 예외 경로에 남아 있는 접근 권한을 확인하지 않음.

## 위협 시나리오

1. 병원 또는 연구기관의 계약 종료가 시작되어 일부 사용자 계정만 먼저 비활성화됨.
2. 기존 refresh token, 활성 데이터셋 전달 경로, export 파일, 생성된 dataset object, cache, 예외 권한 중 일부가 그대로 남아 있음.
3. 오프보딩 대상 사용자는 기존 token이나 기존 다운로드 경로를 이용해 다시 접근을 시도하고, 과거 수신자는 이미 전달된 객체를 재사용함.
4. 결과적으로 계약 종료 이후에도 데이터 접근이나 재다운로드가 가능해지고, 오프보딩 완료 여부 자체를 신뢰하기 어려워짐.

## 필수 예방 사항

- 오프보딩을 `계정 회수`, `세션 폐기`, `데이터 export`, `데이터 파기`, `파기 검증`으로 분리합니다.
- 파기 자산 목록에 S3 객체, DB schema, generated dataset, cache, 활성 전달 권한, backup 식별 정보를 포함합니다.
- 유예 기간과 최종 파기 시점을 문서화합니다.
- 오프보딩 진행 중에는 tenant 상태를 `SUSPENDED` 등으로 전환해 신규 접근을 차단합니다.

## 필수 탐지 사항

- 비활성 tenant의 API 호출을 탐지합니다.
- 잔존 object, orphaned account, 활성 전달 권한을 점검합니다.
- 오프보딩 완료 후 일정 기간 동안 후속 접근 시도를 모니터링합니다.

## 대응 요구사항

- 잔존 artifact를 즉시 삭제 또는 격리합니다.
- 모든 관련 세션과 예외 자격증명을 재검토합니다.
- 오프보딩 증적을 재작성하고 누락 자산을 목록화합니다.

## 검증 기준

- 오프보딩 완료 후 API 호출, dataset 다운로드, 객체 접근이 모두 거부되어야 합니다.
- 파기 증적이 계정, DB, S3, cache, export 단위로 남아야 합니다.
- backup 및 복구본 처리 기준이 별도 문서로 정의되어야 합니다.

## 운영 점검 항목 및 주기

| 증빙 항목 | 최소 내용 | 주기 |
|----------|----------|------|
| 오프보딩 체크리스트 | 계정 회수, 세션 폐기, export 제공, 파기, 검증 단계 | 오프보딩 시마다 |
| 파기 증적 | DB schema, S3 객체, dataset artifact, cache, export 삭제 결과 | 오프보딩 시마다 |
| post-offboarding 차단 테스트 | 기존 계정, 기존 token, 기존 전달 경로로 접근 시도한 결과 | 오프보딩 시마다 |
| 예외 보유 현황 | backup 보존, 법적 보존, 분쟁 보존 등 잔존 사유와 만료 시점 | 월간 |
