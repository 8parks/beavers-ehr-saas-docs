# 시나리오 기반 보안 설계

> 구체적인 비즈니스 로직 시나리오와 아키텍처가 각 흐름을 어떻게 보호하는지 설명합니다.

## 시나리오 목록

| 시나리오 | 핵심 보안 목표 | 관련 위협 |
|---------|--------------|---------|
| [환자 기록 생성 & 서명](./patient-record-creation) | 무결성, 인가, 감사 | T-008, T-001 |
| [환자 기록 조회](./patient-record-retrieval) | 테넌트 격리, 인가, 감사 | T-001, T-002, T-005 |

| [크로스 테넌트 접근 방지](./cross-tenant-prevention) | 테넌트 격리, 접근 제어 | T-001 |

| [보안 탐지 & 자동 대응](./security-remediation) | 구성 드리프트 탐지, 자동 복원 | T-004, T-009 |
| [관리 계정 & 앱 계정 분리 운영](./admin-account-separation) | 계정 경계, 최소 권한 | T-002, T-009 |

## 시나리오 템플릿

각 시나리오는 다음 구조로 작성됩니다.

1. 비즈니스 로직 정의
2. 보안 목표
3. 기능 흐름 (사용자 관점)
4. 아키텍처 흐름 (AWS 서비스 관점)
5. 단계별 상세 흐름
6. AWS 서비스 및 책임
7. 보안 컨트롤
8. 컴플라이언스 매핑
9. 위협 및 완화
10. 설계 체크리스트
11. 미해결 질문
