# 시나리오: 보안 탐지 & 자동 대응

## 1. 비즈니스 로직 정의

AWS Config가 비준수 리소스를 탐지하면 Security Hub, EventBridge, Lambda, SSM Automation을 통해 자동으로 대응하고 보안 운영자에게 알림을 전달하는 흐름입니다.

## 2. 보안 목표

- 보안 이상 및 구성 드리프트를 신속하게 탐지합니다.
- 정의된 유형의 이상 탐지 결과는 자동으로 대응됩니다.
- 모든 대응 활동은 감사 가능합니다.
- 잘못된 자동 대응이 서비스를 중단시키지 않도록 안전 메커니즘이 적용됩니다.

## 3. 자동화 파이프라인

```
AWS Config (비준수 탐지)
    ↓
AWS Security Hub (결과 집계)
    ↓
Amazon EventBridge (이벤트 라우팅)
    ↓
AWS Lambda (전처리, 분류, 대응 예약)
    ↓
SSM Automation Runbook (대응 실행)
    ↓
CloudTrail + CloudWatch (활동 기록)
    ↓
보안 운영자 알림 (SNS, 대시보드, 티켓)
```

## 4. 대응 유형 분류

| 이상 탐지 유형 | 대응 방식 | 예시 |
|--------------|---------|------|
| 즉각 자동 대응 | SSM 런북 자동 실행 | S3 퍼블릭 접근 차단 |
| 승인 후 대응 | 보안 운영자 승인 후 실행 | 보안 그룹 규칙 변경 |
| 알림만 | 티켓 생성, 알람 | 복잡한 구성 변경 |

## 5. AWS 서비스 및 책임

| AWS 서비스 | 역할 |
|-----------|------|
| AWS Config | 리소스 구성 준수 상태 지속 평가 |
| AWS Security Hub | 보안 결과 집계 및 우선순위화 |
| Amazon EventBridge | 이벤트 라우팅 및 규칙 매칭 |
| AWS Lambda | 이벤트 전처리, 분류, 대응 예약 |
| SSM Automation | 승인된 대응 런북 실행 |
| Amazon SNS | 보안 운영자 알림 |
| AWS CloudTrail | 대응 활동 감사 |
| Amazon CloudWatch | 대시보드 및 알람 |

## 6. 보안 컨트롤

| 컨트롤 | 구현 방법 | 관련 위협 |
|--------|---------|---------|
| 구성 드리프트 탐지 | AWS Config 규칙 | T-004 잘못된 구성 |
| 자동 대응 감사 | CloudTrail 기록 | T-009 대응 역할 남용 |
| 최소 권한 대응 역할 | IAM 역할 최소 권한 | T-009 |

## 7. 설계 체크리스트

- [ ] 핵심 AWS Config 규칙이 활성화되어 있습니다.
- [ ] Security Hub 표준이 활성화되어 있습니다.
- [ ] EventBridge 규칙이 보안 결과를 올바르게 라우팅합니다.
- [ ] SSM 런북 IAM 역할이 최소 권한으로 설정되어 있습니다.
- [ ] 모든 자동 대응 활동이 CloudTrail에 기록됩니다.
- [ ] 자동 대응 실패 시 수동 대응 절차가 있습니다.
